## 結論

1. **Hermes self-improvement 存在但效率極低** — 5小時訓練得 1 個真正 SKILL
2. **觸發條件太嚴格** — 需要長時間自然對話，唔適合刻意訓練
3. **大部分 SKILL 係 preset** — 94 個入面 90% 係出廠設定，唔係學返嚟
4. **當普通 CLI chat 用係 OK** — 但唔好期望自主學習會帶嚟顯著價值
5. **唔值得特登去 train** — 費時失事

## 最終結論（May 29 03:00 更新）

### Hermes 自我學習能力真相

經過 5+ 小時全面測試，以下是無修飾的結論：

**1. Self-improvement 機制存在但效能極低**
- 唯一真正由 Hermes 自己創建嘅 SKILL：`supplier-claim-verification` (112行)
- 其餘 4 個「Hermes SKILL」既真正來源：
  - `auto-fix-workflow` — **author: Ally**，即係我寫嘅，唔係 Hermes 自己學
  - `hermes-agent` — **author: Hermes Agent + Teknium**，官方 default preset
  - `skill-audit-workflow` — sub-agent 創建，唔係自我學習
  - `github-pr-workflow` — 官方 default preset

**2. 觸發門檻太高**
- 成功案例：66 messages / 14 分鐘自然對話 → 1 個 SKILL
- 失敗案例：15 turns 自然對話 → 0 個 SKILL
- 機械式 Q&A（100-turn script）→ 完全無效

**3. 實際用途**
- 當普通 CLI chat client：✅ 合格
- 期待自主學習變聰明的 AI agent：❌ 效率太低，唔值得特登 train

### 關於「94 個 SKILL」的真相

之前覺得 Hermes 有 94 個 SKILL 好似好勁，但實際上：
- ~90% 係安裝時自帶的 default/preset SKILL（來自 Teknium 社群）

- 而且自我學習產生的質量參差
