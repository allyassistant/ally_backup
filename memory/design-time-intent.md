# 時間意圖自動捕獲 - 設計方案

## 目標
當用戶講「明天發」、「後日做」等時間意圖時，自動捕獲並加入 Apple Reminders。

## 為乜揀 Apple Reminders？
- 原生 iOS/Mac 集成，手機同 Mac 同步
- 支援重複提醒、位置提醒
- 可用語音 Siri 控制
- 已有 `remindctl` 命令工具

## 觸發詞列表

### 直接時間詞（立即觸發）
| 觸發詞 | 範例 | 轉換動作 |
|--------|------|----------|
| 聽日 / 明天 / 明日 | 「聽日發俾佢」 | +1 day, 09:00 |
| 後日 / 大後天 | 「後日做」 | +2/+3 day, 09:00 |
| 聽朝 / 聽早 / 聽日上午 | 「聽朝九點傾」 | +1 day, 09:00 |
| 今晚 / 今晚八點 | 「今晚八點開會」 | Today, 20:00 |
| 晏啲 / 下午 | 「晏啲覆佢」 | Today, 14:00 |
| 聽晚 / 聽晚 | 「聽晚七點」 | +1 day, 19:00 |
| 禮拜X / 星期一~日 | 「禮拜三傾」 | Next weekday, 10:00 |
| 下週 / 下星期 | 「下星期交」 | Next week same weekday |
| 月尾 / 月頭 | 「月尾結帳」 | End/Start of current month |
| 下個月 | 「下個月先整」 | 1st of next month |

### 相對時間詞（需配合上下文）
| 觸發詞 | 範例 | 轉換動作 |
|--------|------|----------|
| 幾日後 | 「幾日後覆」 | +3 days |
| 一個星期後 | 「一星期後再傾」 | +7 days |
| 半個月後 | 「半個月後check」 | +14 days |
| 幾時...？（查詢） | 「幾時得閒？」 | 只回覆，不設提醒 |

### 任務動作詞（需配合時間）
| 觸發詞 | 範例 |
|--------|------|

| 覆 / 覆佢 | 「聽日覆張生」 |
| 傾 / 傾偈 / 傾生意 | 「聽晚傾」 |
| 整 / 整好 | 「聽日整好份報價」 |
| 交 / 提交 | 「聽日交貨」 |
| 跟進 / 跟进 | 「後日跟進」 |
| 開會 / 會議 | 「禮拜三開會」 |

## 處理流程

### 流程圖
```
用戶輸入
    ↓
[意圖檢測模組]
    ↓
係時間意圖？ ──否──→ 正常處理
    ↓係
提取關鍵元素：
  - 時間 (when)
  - 動作 (what)
  - 對象 (who/which)
    ↓
[時間轉換引擎]
    ↓
計算提醒時間
    ↓
[Apple Reminders API]
    ↓
建立提醒 + 回覆確認
```

### 演算法：時間提取

#### Step 1: 正則匹配
```javascript
// 匹配時間關鍵詞
const TIME_PATTERNS = {
  '聽日|明天|明日': { unit: 'day', value: 1 },
  '後日': { unit: 'day', value: 2 },
  '大後天': { unit: 'day', value: 3 },
  '聽朝|聽早': { unit: 'hour', value: 9 },
  '今晚': { unit: 'hour', value: 20 },
  '晏啲': { unit: 'hour', value: 14 },
  '禮拜一|星期一': { unit: 'weekday', value: 1 },
  '禮拜二|星期二': { unit: 'weekday', value: 2 },
  // ...
};
```

#### Step 2: 上下文推斷
- 如果淨係講「聽日」但冇講時間，預設 09:00
- 如果講「聽日晏啲」，推斷為 14:00
- 如果講「聽晚」，預設 19:00

#### Step 3: 動作 + 對象提取
```javascript
// 匹配動作
const ACTION_PATTERNS = {
  '發|寄': '發送',
  '覆|覆覆': '回覆',
  '傾|傾偈': '會議/傾談',
  '整|整好': '完成',
  '交|提交': '提交',
  '跟進': '跟進',
  '提醒': '提醒'
};

// 匹配對象
// - 人名：從用戶歷史或內置列表提取
// - 事項：從上下文提取關鍵詞
```

## 輸出格式

```

• 發送 Email 俾 Josh (來自對話)
• 回覆 Desanna (來自對話)
• 跟進報價單 (來自對話)

```

### 回覆用戶示例
```
✅ 已設定提醒：
📅 聽日 (2月17日) 09:00
📝 發送報價俾客戶
```

## Apple Reminders 整合

### 使用 remindctl
```bash
# 添加提醒
remindctl add --title "發送報價俾張生" --due "2026-02-17 09:00"

# 查看今日提醒
remindctl today

# 完成提醒
remindctl complete <id>
```

### 列表分類
建議使用以下列表：
- **AI Task** - AI 系統自動創建既任務
- **個人** - 用戶私人事項
- **工作** - 工作事項

## 強制分析模式整合

### 觸發條件
時間意圖檢測應該喺 **強制分析模式** 之前執行：
1. 首先檢測時間意圖
2. 如果係，建立提醒
3. 然後進入強制分析模式正常處理

### 優先級
- 時間意圖捕獲：**高優先級**（確保唔漏）
- 強制分析：**標準處理**
