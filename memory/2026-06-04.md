### 關鍵事實 — Artifact 2: model_router.js Architecture

**Interface signature**:
```js
async function routeModel({ text, route, context = {} }) → Promise<{
  provider, model, baseUrl, apiKey, timeout, extraBody, fallbackChain, decisionId
}>
```

**7-step key logic** (pseudo-code):
1. `config = loadRouteModelYaml()` — mtime-aware cache, 唔每次 hit disk
2. `validateRouteConfig(config)` — fail-fast if schema broken
3. `routeCfg = config.routes[route]` — if missing → throw "Unknown route: X"
4. `chain = [primary.provider, ...fallback_chain]` — dedup, keep order
5. `resolved = failure_recovery.resolveProvider(chain)` — first healthy, or 'none' if all unhealthy
6. `return mergeResolvedConfig(resolved, routeCfg, config.providers)` — merge provider defaults + route overrides + env resolution
7. `async appendDecisionLog({...})` — NON-blocking disk write (fires Promise, caller 唔 await)

**3 個 integration points**:
1. **classifier.js call site** (line 74 `regexClassify` return 後): 加一行 `await routeModel({text, route, context})`
2. **route-enforcer system prompt inject**: 由 `[ROUTER: X]` 變 `[ROUTER: X] [MODEL: provider/model] [FALLBACK: a,b,c]`
3. **decision_log.jsonl append**: additive 新 fields (suggestedModel, suggestedProvider, actualProvider, fallbackDepth, latencyMs, success, costEstimate, schemaVersion) — 863 條 baseline 完全 preserve

### 關鍵事實 — Artifact 3: failure_recovery.js Architecture

**Health check loop 設計**:
- `health_cache = Map<provider, {healthy, lastCheck, failureCount, cooldownUntil}>`
- 60s interval background loop
- `failure_threshold = 3` (連續 3 次失敗先 mark unhealthy, 避免 single-false-positive)
- `max_concurrent_probes = 3` (semaphore 防止 stampede)
- `probe_method = list_models` (cheapest call)
- `recovery_probe_interval = 30s` (失敗 cooldown 後 30s probe 一次)

**resolveProvider logic**:
```
for provider in chain:
  entry = health_cache.get(provider)
  if entry.healthy AND entry.cooldownUntil < now:
    return provider
return 'none'  // all unhealthy
```

**onCallFailure 處理**:
- `entry.failureCount++`
- if `failureCount >= 3`: `entry.healthy = false`, `cooldownUntil = now + 60s`
- schedule recovery probe (30s 後 ping 一次)
- 失敗 probe 自動延長 cooldown (+30s)

**Race condition 處理** (concurrent sub-agent spawn):
- **versioned stamp pattern**: 每個 `routeModel` call 拎 `decisionId = UUID()` 寫入 log
- 失敗時 `onCallFailure(provider, decisionId, error)` 接收 identifier — 同一 provider 嘅 concurrent failure 自動 debounce (同一 `cooldownUntil` 唔會重覆覆寫)
- Health check loop 用 semaphore 防止 probe storm
- `chain.shift()` 喺單 thread Node.js 唔係真問題, 但 log 方便 debug

**Integration with model_router.js**: pull model 變 push model — 每次 `routeModel` call 同時 fire `failure_recovery.observeCall(provider, latencyMs, success)`, 自動 update cache, caller 完全唔需要 explicit health check call
