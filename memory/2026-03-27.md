## 🔧 L0/L1 Timeout 解決方案（2026-03-27 02:47 HKT 更新）

### 問題分析
- agentTurn 模式自動載入所有 workspace context + sessions
- Sessions 目錄 50MB，最大 5.5MB
- Input tokens 爆升至 304k（正常 10-50k）
- TOOLS.md (14KB) + AGENTS.md (8KB) 只佔 2%，sessions 先係元凶

### 解決方案：systemEvent + AI API 直接生成
- **新 Script**: `scripts/l0_l1_generator.js`
- **原理**: systemEvent 只執行 script，script 直接 call AI API
- **好處**: 
  - 只讀取昨日 L2 (memory/*.md)
  - 唔載入 workspace context
  - 唔載入 sessions
  - Token 大幅減少

### Cron Jobs 已更新
| Job | ID | 舊模式 | 新模式 |
|-----|-----|--------|--------|
| L0 @ 00:05 | a240652b... | agentTurn (Kimi) | systemEvent |
| L1 @ 00:35 | b66f9f8f... | agentTurn (Kimi) | systemEvent |

### 新流程
```
00:05 → systemEvent → l0_l1_generator.js
                        ↓
                   只讀取昨日 L2
                        ↓
                   Call AI API (MiniMax)
                        ↓
                   生成 L0 + L1
```

### MiniMax vs Kimi
- **MiniMax**: ✅ 有直接 API，可以從 script 調用
- **Kimi**: ❌ 冇直接 API，必須透過 agentTurn
