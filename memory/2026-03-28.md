## 💡 學到既嘢

1. **預防勝於治療** — 大量問題係因為冇 systematic review，以後要定期做 code review
2. **Security 容易被忽略** — Shell injection、密碼明文等，雖然「小事」但係風險
3. **時區問題常見** — UTC vs HKT 搞到好多 bug，以後要统一用 HKT
4. **架構清晰好過數量** — 169 個 scripts 但係 60+ 係 legacy，清理後更易維護
5. **CRON job 文件要同步更新** —好多功能有定時執行但冇在 crontab 列出，難以審計
