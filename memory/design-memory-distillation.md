# 記憶提煉機制 - 設計方案

## 目標
自動從每日日誌（daily notes）提煉重要內容到 MEMORY.md，形成長期知識積累。

---

## 提煉頻率
- **標準**：每週一次（建議禮拜日或禮拜一）
- **觸發**：每週日晚 heartbeat 時執行
- **手動**：用戶可隨時要求「提煉記憶」

---

## 乜嘢係「重要內容」？

### 分類標準

#### 🔴 高優先級（必須提煉）
| 類型 | 範例 | 原因 |
|------|------|------|
| **業務決策** | 「決定用 XX 定價策略」「今後唔再...」 | 影響業務方向 |
| **重要變更** | 「Rapaport 算法更新」「Shape 簡寫修正」 | 系統性改動 |
| **錯誤教訓** | 「Shape PR/PS 不要再搞混」「不要用 short form MiniMax」 | 防止重複犯错 |
| **客戶偏好** | 「張生偏好 RBC」「王太想要 1.5-2ct」 | 服務質量 |
| **新知識** | 「學識咗 XLOOKUP」「發現新供應商」 | 技能積累 |

#### 🟡 中優先級（應該提煉）
| 類型 | 範例 | 原因 |
|------|------|------|
| **約定/承諾** | 「答應禮拜三覆張生」「答應星期日交貨」 | 防止漏嘢 |
| **待辦事項** | 「跟進庫存盤點」「聯繫新工廠」 | 工作追蹤 |
| **價格資訊** | 「呢批貨成本 XX」「最新報價 XX」 | 業務參考 |
| **市場觀察** | 「最近呢種 Shape 幾好賣」「客戶傾向 XX」 | 趨勢分析 |

#### 🟢 低優先級（可選提煉）
| 類型 | 範例 | 原因 |
|------|------|------|
| **一般對話** | 「傾咗啲咩」「問咗啲咩」 | 歷史記錄 |
| **重複資訊** | 「又一次被問 XX」「又出現 XX 問題」 | 模式識別 |

## 提煉流程

### Step 1: 收集（每週日）
```bash
# 收集過去 7 日既 daily notes
# memory/2026-02-09.md ~ memory/2026-02-15.md
```

### Step 2: 分析
使用 Kimi/MiniMax 分析每篇日記：
1. **識別重要內容**（按上述分類）
2. **評估價值**（高/中/低）
3. **去重**（避免重複記錄）
4. **分類**（業務決策/錯誤教訓/新知識/待辦/...）

### Step 3: 提煉
將高優先級 + 部分中優先級內容：
1. 寫入 MEMORY.md 相關章節
2. 去相同主題章節合併
3. 去除已過時既內容

### Step 4: 存檔
1. 將原始 daily notes 移去 `_archive/`
2. 更新 `memory-index.json`
3. 記錄提煉日誌

## 三層記憶整合

### 提煉後既內容分布

| 層級 | 內容類型 | 存放位置 |
|------|---------|---------|
| **情景記憶** | 具體事件、對話、決定 | `memory/_daily/` 或 `_archive/` |
| **語義記憶** | 原則、知識、規則 | `MEMORY.md` 主要章節 |
| **強制規則** | 重要警告、禁止事項 | `MEMORY.md` 頂部優先顯示 |

### 提煉時既分類邏輯
```
IF 包含 "不要再" 或 "記住" 或 "禁止"
   → 強制規則層
ELSE IF 包含 "決定" 或 "今後" 或 "政策"
   → 語義記憶層 (業務決策)
ELSE IF 包含 "學識" 或 "發現"
   → 語義記憶層 (新知識)
ELSE
   → 情景記憶層 (保留原始)
```

## 強制分析模式整合

### 觸發時間
- **自動**：每週日 heartbeat 時
- **手動**：用戶講「提煉記憶」

### 執行模型
- **主要分析**：Kimi（高價值判斷）
- **結構整理**：MiniMax（格式統一）
- **執行操作**：Qwen3（檔案搬運、JSON 更新）

### 優先級
- 低於用戶對話
- 使用閒置時段執行

## Kimi 實施注意點

1. **避免重複**：提煉前檢查 MEMORY.md 是否已有類似內容
2. **時間標記**：每次更新加日期標籤（2026-02-16 更新）
3. **適度提取**：唔好乜都拎，保持 MEMORY.md 精簡
4. **三層分流**：根據內容性質分流到唔同層次
5. **保持可讀**：MEMORY.md 仍然需要人類易讀

## 維持記憶質量既方法

### 定期 review
- 每週提煉後快速瀏覽 MEMORY.md
- 移除過時（>6個月）既內容
- 合併重複章節

### 質量標準
- 每條記憶都要有日期標籤
- 避免過長既章節（>500字考慮濃縮）
- 優先保留「點做」而非「發生咩事」
