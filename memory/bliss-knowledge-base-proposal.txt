## 2. 文章核心內容摘要

### 三層記憶模型
- **Tier 1 (信息層)**: 原始記錄、學習筆記、對話記錄
- **Tier 2 (知識層)**: 每日提煉、工作日誌、提煉後既知識點
- **Tier 3 (智慧層)**: 長期記憶、跨領域洞察、核心方法論

### 三個認知升級
1. **本地記憶 → 雲端知識中台**: 有道雲筆記 × OpenClaw MCP協議
2. **被動存儲 → 主動加工**: Skill 自動提煉文章成知識層筆記
3. **單點工具 → 系統基礎設施**: 多 Agent 共享同一套知識庫

## 4. 我諗既可行方案

### 方案 A: 加強 Weekly Correction Loop (短期)
**目標**: 自動從錯誤 + 對話記錄提煉「核心方法論」

**具體做法**:
1. 每周日 18:00 運行
2. 讀取 errors.json + 當周 L0/L1
3. AI (MiniMax) 分析重複錯誤模式
4. 自動生成「規則草案」
5. 寫入 AGENTS.md P0 區域
6. Discord 通知確認

**預期效果**: 將 Tier 1/2 提煉為 Tier 3 智慧層

---

### 方案 B: 建立 Shared Core Knowledge (中期)
**目標**: Ally 同 Bliss 真正共享核心知識

**具體做法**:
1. 創建 `memory/core-knowledge.json`
2. 內容包含:
   - 系統架構 (HA, Failover)
   - 重要決策記錄
   - 錯誤教訓 (P0 級別)
   - 用戶偏好
3. 兩邊機器都讀取同一個檔案
4. 重要更新時自動同步

**預期效果**: 跨 Agent 知識一致性提升

---

### 方案 C: 晨間簡報 Morning Brief (簡單)
**目標**: 每日早上主動推送當日重點

**具體做法**:
1. 新增 Cron Job: 每日 09:00
2. 讀取:
   - 今日 Reminders (from Apple Reminders)
   - 當日到期 Issues
   - 昨日 Daily Summary 重點
3. 用 Discord Bot API 發送到 #⚙️系統
4. 格式: 簡短、行動導向

**預期效果**: 更主動既信息推送 (類似文章講既 09:00 簡報)

---
