## 🚨 Joshua 審核發現的 Fancy Color 解析問題

### 問題分類
| 問題類型 | 報告數 |
|---------|--------|
| `fancy shapes like XXX` | 5個 |
| `Fancy` (單詞) | 2個 |
| `fancy shapes` | 1個 |

### 根本原因 (Kimi Code CLI + MiniMax 代碼審計)

1. **extractFancyColor() P1** - 接受 "Fancy" alone (無 intensity level)
2. **"fancy shapes like XXX" P1** - Shape name 被當作 color 提取
3. **形狀關鍵詞 P1** - Pear/Cushion/Round 可能被錯誤識別為 color
4. **Recovery logic P2** - 只在 calculateClawScore 運行，parseGIACertificateText 不修復

### 修復建議 (已生成)

| 優先級 | 修復位置 | 建議 |
|--------|---------|------|
| P1 | extractFancyColor() | 必須有 intensity keyword |
| P1 | normalizeData() | 形狀關鍵詞保護 |
| P2 | parseGIACertificateText() | 加入 recovery logic |
| P1 | parseGIACertificateText() | 修復 "fancy shapes like" 腐敗檢測 |

## 關鍵學習

### Fancy Color Regex Bug
- `extractField(text, /color\s*grade[:\s]*([D-Z](?:\s*-\s*[A-Z])?)/i)` 會匹配 "fancy" 的 'f'
- 解決方案：`(?<![a-z])` + `(?![a-z])` negative lookahead/lookbehind

### Intensity Pattern
- "Fancy" alone 需要 intensity keyword (intense/vivid/deep/dark/light/moderate)
- 沒有 intensity = corrupted 或 plain text
