## SSH Security Upgrade (Afternoon)

### Problem
- HA scripts used password-based SSH with hardcoded credentials
- Security risk: passwords exposed in scripts
- Used expect/sshpass for automation (insecure)

### Solution Applied

#### 1. SSH Key-Based Authentication
- Generated ED25519 SSH key pair on both machines
- Configured bidirectional key auth (Ally ↔ Bliss)
- Removed password-based authentication

#### 2. Disabled Password SSH Login
- Modified `/etc/ssh/sshd_config` on both machines
- Set `PasswordAuthentication no`
- Now only SSH key authentication allowed

#### 3. Updated Scripts
- `failover_detector.sh` - Removed sshpass/expect, use SSH key
- `ha_coordinator_ssh.sh` - Removed expect script, use SSH key
- `ha_coordinator_tailscale.sh` - Removed sshpass, use SSH key

#### 4. Security Improvements
| Before | After |
|--------|-------|
| Password-based auth | SSH Key (ED25519) |
| expect/sshpass | Native SSH |
| Loose host checking | StrictHostKeyChecking=yes |
| Password login enabled | Password login disabled |

### Files Updated
- `~/.ssh/id_ed25519` (Ally)
- `~/.ssh/authorized_keys` (both)
- Scripts in `~/.openclaw/workspace/scripts/`
