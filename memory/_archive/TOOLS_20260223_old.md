### Apple Notes (AppleScript) - UPDATED 2026-02-01

**重要**: Apple Notes 需要用 HTML `<br>` 換行，唔係 `\n`

**創建格式良好嘅 Note**:
```bash
osascript << 'EOF'
tell application "Notes"
    set theFolder to folder "Ally's Notes"
    set noteTitle to "AI 每日總結 - 2026-02-01"
    set noteBody to "<h2>📋 今日工作總結</h2>" & "<br><br>" & \
        "<b>主要成就:</b><br>" & \
        "• 完成項目 A<br>" & \
        "• 修復問題 B<br><br>" & \
        "<b>明日計劃:</b><br>" & \
        "• 繼續優化"
    make new note at theFolder with properties {name:noteTitle, body:noteBody}
end tell
EOF
```

**HTML 格式指南**:
| Markdown | Apple Notes HTML |
|----------|------------------|
| `# 標題` | `<h1>標題</h1>` |
| `## 標題` | `<h2>標題</h2>` |
| `**粗體**` | `<b>粗體</b>` |
| 換行 | `<br>` |
| 列表項 | `• 內容<br>` |
| 分隔線 | `<br>---<br>` |

**舊方法** (保留參考):
- `memo notes -f "Folder" -a` (opens Vim, not fully automated)

### Browser Usage (2026-02-18) - UPDATED 2026-02-21
- **Always close browser after use**: Use `browser action=stop` to free resources
- This rule applies to ALL browser usage (not just RapNet check)
- After completing any task that uses browser, ALWAYS close it
- **Profile Selection (2026-02-21):**

- Used for: To-do items, self-reminders, task tracking

### MiniMax Sub-agent Model Format (CRITICAL!)
**2026-02-16: Don't use short form!**

**When spawning MiniMax sub-agents, MUST use full format:**
```
minimax-portal/MiniMax-M2.5
```

**DON'T use:**
```
minimax                    ❌ (causes "model not allowed: kimi-coding/minimax")
minimax-portal/minimax     ❌
MiniMax-M2.5               ❌
```

**Why it matters:**
- Short form `minimax` → system uses default provider `kimi-coding` → `kimi-coding/minimax` → error
- Full form → correct provider `minimax-portal` → works correctly

**Verified working sessions:**
- `MiniMax-Roundtable-Analysis` ✅
- `MiniMax-Test-Now` ✅

### 💎 Diamond Technical Reference (From MEMORY.md - 2026-02-20)

**Rapaport 價格計算:**
1. **形狀**: Round=RBC 表,其他=Pear 表
2. **重量**: 1.00-1.49 | 1.50-1.99 | 2.00-2.99 | 3.00-3.99 | 4.00-4.99 | 5.00+
3. **重要規則**: 大過5卡都用5.00-5.99表計
4. **計算**: (表格值×100 × Carat) × (1-Discount%)

**Rapaport 估值參考:**
| 等級 | Rapaport 折扣 |
|------|---------------|
| 🔴 偏低 | -35% ~ -45% |
| 🟡 合理 | -20% ~ -35% |
| 🟢 偏高 | -10% ~ -20% |
| 💎 極高 | 0% ~ +10% |

**V9 Stock List 要求:**
- 列: Parcel Name | Shape | Crt | Color | Clarity | Cut | Pol | Symm | Measur | Depth | Table | Fluor | Lab | Cert No | Memo Price
- 排序: Shape (RBC先) → Carat (大→小) → Color (D→Z)
- 格式: 全部置中 | Crt 2位小數 | 標題粗體

**庫存查詢政策:**
- 用廣東話回覆
- 嵌入英文鑽石資料
- 預設不顯示Memo Price
- 不要單獨發WhatsApp
