## 原因

`/reset` 係 **OpenClaw Admin 指令**，需要特定權限。

## 而你家既狀況

從之前既 OpenClaw status，我見到：

```
WARN Discord allowlist contains name or tag entries
Found: channels.discord.guilds.1473343323124011182.users:joshua_chan0209
```

## 可能問題

| 問題 | 說明 |
|------|------|
| **Name matching** | Allowlist 用緊 username (`joshua_chan0209`) 而唔係 Discord ID |
| **Normalized slugs** | OpenClaw 將 username 轉換後可能唔 match |
| **指令限制** | `/reset` 可能淨限特定 admin 用 |

## 解決方案

**方案 1：用 Discord ID（推薦）**
```bash
openclaw config set channels.discord.guilds.1473343323124011182.users "749642704711057509"
# 用你既 Discord User ID 而唔係 username
```

**方案 2：暫時用 CLI**
```bash
# 喺 terminal 執行
openclaw reset
```

**方案 3：加 `dangerouslyAllowNameMatching`**
```bash
openclaw config set channels.discord.dangerouslyAllowNameMatching true
```
