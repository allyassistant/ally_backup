**Topic**: Default model vs current model differences

**Key Points**:

- `/model` command only affects current session
- Each session runs independently with its own model
- Cron jobs and sub-agents may use different models simultaneously

**User switched to Qwen3**: Using `ollama/qwen3:14b` for local processing

### Health Monitor Notification Fix
**Problem**: User receiving WhatsApp notifications every 5 minutes from health monitor

**Root Cause**: 
- Cron job using `systemEvent` triggering main session relay
- Script output being sent regardless of status

**Solution Applied**:
1. Modified health_monitor.js to only show reports when issues exist
2. Changed cron job from `systemEvent` to `isolated` agent session
3. Added instruction: Only send WhatsApp when status !== 'up'
4. Normal operation = silent, issues = alert

**Result**: User will only receive notifications when system has problems
