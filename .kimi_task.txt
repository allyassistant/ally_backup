## 背景
P0-1 isProtectedByTry 已完成。現在需要實施 Skip List 機制，讓用戶可以將確認為 false positive 的問題標記為跳過，之後審計時自動忽略。

## 需要實現的功能

### 1. 新增 Skip List 狀態檔案
- 位置：.state/auto_fix_skip_list.json（使用 WS 和 STATE_DIR）
- 格式：
{
  "skips": [
    {
      "ruleId": "HR-001",
      "file": "scripts/pattern_archive.js",
      "lines": [54, 72, 92, 119],
      "markedAt": "2026-04-04T02:00:00.000Z",
      "reason": "confirmed-false-positive"
    }
  ],
  "version": 1
}

### 2. 新增函數（放在現有 helper functions 附近）

#### loadSkipList()
- 讀取 .state/auto_fix_skip_list.json
- 如果檔案唔存在，返回空的 skip list {skips: [], version: 1}
- 使用 try-catch 處理錯誤

#### saveSkipList(skipList)
- 寫入 .state/auto_fix_skip_list.json
- 使用 atomic write（tmp + rename）
- isDryRun 模式唔實際寫入

#### markFalsePositive(ruleId, file, lines, reason = 'confirmed-false-positive')
- 將問題標記為 false positive
- 加入 skip list（如果已存在則更新 lines）
- 自動保存

#### isSkipped(issue, file)
- 檢查問題係咪喺 skip list 中
- 匹配 ruleId + file + lines
- 傳入的 issue 格式: { rule: ruleId, lines: [54, 72], ... }
- 檔案格式: result.file (相對路徑)

### 3. 整合到 analyzeFile()
在 High-risk 檢測後（for loop 完成後），加入過濾 skip list 的邏輯：
result.highRisk = result.highRisk.filter(issue => {
  if (isSkipped(issue, result.file)) {
    return false;
  }
  return true;
});

### 4. 新增 CLI 命令
在 main() switch 裏面加入 skip case：
- skip --list：顯示現有 skip list 內容
- skip --rule=HR-001 --file=scripts/test.js --lines=45,46,47：添加 skip list 項目

注意：需要從 report.highRisk 搵到對應的 rule ID actual name（如 eval-usage 而唔係 HR-001）
