## 核心 Scripts 詳解

### 🔀 Router 系統（Message Routing Pipeline）

```
Message Received
       ↓
[hooks/message_received.js] ───→ auto_remember trigger detection
       ↓
[router/classifier.js] ───→ Fast regex-based routing decision (~1ms)
       ↓
/router/decision.json ← written by classifier
       ↓
[auto-router/handler.js] ───→ Python router.py hook (sub-agent spawning)
       ↓
Route Dispatched → SPAWN / DIRECT_ANSWER / SOP / CODE / NONE
```

| Script | Purpose | Key Config |

| `auto-router/handler.js` | Hook handler: runs router.py → writes `.router-decision.json` | — |
| `auto_skill_router.js` | Intent detection + Skill matching + auto-execution | Skills registry |

**Route Labels**: `FDQ` / `DIRECT_ANSWER` / `SOP` / `SPAWN` / `CODE` / `BROWSER` / `NONE`

### 🟡 Quality 管理（CQM 生態）

| Script | Purpose | CLI |
|--------|---------|-----|
| `code_quality_manager.js` | Unified entry: FileDiscovery + IssueAggregator + AuditOrchestrator | `node scripts/code_quality_manager.js scan\|verify\|repair` |
| `verify_edit.js` | **Post-Edit 驗證器**：syntax check + P0 violations + undefined fn refs | `node scripts/verify_edit.js <file>` |
| `auto_fix.js` | Auto-audit: scan → auto-fix low-risk → report high-risk for human confirm | `node scripts/auto_fix.js fix\|scan\|report` |
| `anomaly_monitor.js` | Monitors cron output + error frequency + runtime vs historical baselines | `node scripts/anomaly_monitor.js --dry-run` |

**Enforcement Chain**:
1. **Post-Edit** (immediate): `verify_edit.js <file>` → syntax + P0
2. **Pre-Commit** (git hook): CQM scan → block high/critical
3. **Scheduled** (cron): 10:00/15:00/22:00 → full sweep via `code_quality_manager.js`

**P0 Rules**:
- No log double-declaration, sync fs in async, undefined function calls
- execSync/fs/crypto/require wrapped in try-catch
- Surgical changes only — no scope creep

## Pipeline 流程圖

### Message → Response Pipeline

```
[Discord/WebChat Message]
         ↓
[hooks/message_received.js]
  ├── auto_remember trigger → detectAndCreate (issue creation)
  └── classifySync (router/classifier.js) → route label
         ↓
[Router Decision] → .router-decision.json written
         ↓
┌────────────────────────────────────────┐
│ Route Dispatch:                        │
│   FDQ → ask clarifying questions       │
│   DIRECT_ANSWER → answer immediately   │
│   SOP → run SOP flow                   │
│   SPAWN → spawn sub-agent (MiniMax M3) │
│   CODE → pipeline tier judgment        │
│   BROWSER → browser open → snapshot    │
│   NONE → Ally judgment (no spawn)       │
└────────────────────────────────────────┘
         ↓
[Response via message tool or direct reply]
```

### Daily Maintenance Pipeline

```
[04:00 Daily Cron]
         ↓
[daily_maintenance.js]
  ├── Parallel: memory_section_cleanup.js + session_cleanup.js
  └── Sequential: issue_auto_followup.js all
         ↓
[Reports written to memory/]

[06:00 + 18:00]
         ↓
[anomaly_monitor.js]
  ├── Load baseline metrics
  ├── Scan cron output, error frequency, runtime
  └── Alert if anomalous → Discord #⚙️系統
```

## 新增 Script 最佳實踐

### 1. Naming Convention

```
<script_category>_<descriptive_name>.js
```

✅ `stock_updater.js`, `mail_monitor.js`, `heartbeat_recall.js`
❌ `update.js`, `monitor.js`, `run.js`

### 2. Header Template

```javascript
#!/usr/bin/env node
/**
 * <Script Name> - <Brief Purpose>
 *
 * <Longer description if needed>
 *
 * 用法：
 *   node scripts/<script_name>.js              # default
 *   node scripts/<script_name>.js --quiet      # silent (for cron)
 *   node scripts/<script_name>.js --dry-run    # preview only
 *   node scripts/<script_name>.js --json       # JSON output
 *
 * 作者：Ally (<YYYY-MM-DD>)
 */

const _quiet = process.argv.includes('--quiet');
const log = (...args) => { if (!_quiet) console.log(...args); };
```

### 3. Required Imports (use lib/ for shared)

```javascript
const { HOME, MEMORY_DIR, WS } = require('./lib/config');
const { getHKTDate, getHKTDateTime } = require('./lib/time');
const { atomicWriteSync } = require('./lib/state');
```

### 4. CONFIG Extraction (P1 - Magic Numbers → Named Constants)

```javascript
const CONFIG = {
  MAX_RETRIES: 3,
  RETRY_DELAY_MS: 2000,
  OUTPUT_FILE: path.join(MEMORY_DIR, 'output.json'),
  THRESHOLD: 0.85,
  FETCH_TIMEOUT_MS: 30000,
};
```

### 5. Error Handling

```javascript
try {
  const data = fs.readFileSync(filePath, 'utf8');
} catch (e) {
  log(`⚠️ Read failed: ${e.message}`);
  return null;
}
```

### 6. CLI Arguments

```javascript
const _quiet = process.argv.includes('--quiet');
const _dryRun = process.argv.includes('--dry-run');
const target = process.argv[2];  // positional arg
```

### 7. Exit Code Convention

```javascript
// 0 = clean / success
// 1 = issues found (for verify_edit, auto_fix scan)
// process.exit(code)
```

## Cron 整合指南

### Adding a New Cron Job

1. **Via crontab** (manual):
```bash
crontab -e
# Add: 0 6 * * * cd ~/.openclaw/workspace && node scripts/your_script.js --quiet
```

2. **Via openclaw cron** (preferred):
```bash
openclaw cron add \
  --name "Your Script" \
  --schedule "0 6 * * *" \
  --command "node scripts/your_script.js --quiet"
```

3. **Via scheduler.js** (Qwen3 AutoOps):
```bash
node scripts/autoops/scheduler.js
```

### Cron Log Locations

- Main cron log: `~/.openclaw/workspace/.cron_log`
- Mail monitor log: `/tmp/mail_monitor.log`
- Hook fired log: `/tmp/hook_fired.log`
