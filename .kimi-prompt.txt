## 2. Cooldown Logic
- 60s cooldown after failure
- Cooldown 期間 short-circuit return `false` from `isProviderHealthy`
- 過咗 cooldown → 進入 'recovery' state, 允許 1 probe
- Probe success → healthy, failure_count = 0
- Probe failure → cooldown 延長多 60s

## 3. Race Condition Handling
- Use `Map<providerName, Promise>` for in-flight probes
- 如果 probe 進行中, await 個 existing promise (唔重新 probe)
- 避免 thundering herd (多個 concurrent calls 同時 probe)
