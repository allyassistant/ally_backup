## Workflow

1. **收到 X link 或 article** — 立即執行標準分析流程，唔好因為 topic 熟悉就跳步

2. **提取必要資訊**
   - 作者背景（name, handle, bio, followers, expertise）
   - Engagement metrics（views, likes, bookmarks, reposts）
   - Bookmark rate = bookmarks / views × 100%（高於 0.5% 代表高價值內容）
   - 核心觀點（3-5 點，每點一句話）

3. **撰寫 Summary（嚴格格式）**
   - 1句話總結置頂
   - 作者背景
   - 3-5 個觀點（每點一句）
   - Bookmark rate 分析
   - Connect 返系統內相關 topic（如有）

4. **直接 post 做 reply** — 唔好喺 reply 入面加任何 meta commentary

## 禁止事項

❌ **唔准加**：直接 post 個 summary 做 reply，唔加任何：
- 「Obsidian note update ✅」
- 「Done!」 
- 「跟上篇一樣格式」
- 任何 meta 描述你做咗乜

❌ **唔准跳格式** — 唔好因為：
- Topic 似曾相識（「Hermes 我知」）
- Author 係之前見過
- 個 format 「大家都知」
就 cut corners 求其寫幾個 bullet

❌ **唔准重複自己** — 如果個 topic 之前已經有 summary，connect 返之前嘅，唔好 copy paste

## 熟悉度偏見陷阱

當遇到熟悉嘅 topic，好容易 skip 格式步驟。但：
- 用戶 share 條 link 就係想睇你嘅 analysis
- 你嘅 analysis 同 topic 熟唔熟無關
- 格式係 delivery quality，唔係 content quality

正確心態：「呢個 topic 我完全冇見過，我會點做？」→ 跟住個流程做

## 用戶風格偏好

- **語言**：繁體中文 + 廣東話口語混合
- **格式**：Discord summary = 1句總結 + 作者背景 + 3-5 觀點 + bookmark rate + connect topics
- **長度**：視乎 content complexity，但一般 300-800 字
- **風格**：直接、简洁、口語化，唔要官腔
