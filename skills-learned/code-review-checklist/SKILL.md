## Workflow

### Phase A — Spawn Parallel Auditors
1. **Identify target skills** — list all skills in `skills-learned/` to audit. Filter by `status: draft` or recently patched.
2. **Spawn parallel M3 auditors** — for 3+ skills, spawn M3 (thinking:high) sub-agents in parallel, one per skill. Each gets the full audit checklist.
3. **Wait for results** — yield to sub-agents; do not busy-poll. Collect results when they auto-announce.

### Phase B — File Integrity Audit (per skill)
4. **Frontmatter parse** — read SKILL.md, verify `---` delimiters are balanced and YAML frontmatter is valid. Check required fields: `name`, `description`, `status`, `source`, `provenance`, `generatedAt`.
5. **Command examples** — find every bash/node/CLI command block. For each, run `bash -n <command>` or `node --check <script>` to verify syntax. Flag any command that fails parse.
6. **Formula verification** — if the skill contains mathematical formulas, spot-check by substituting known values. Flag any formula that produces wrong output.
7. **Wikilink resolution** — scan for `[[skill-name]]` references. For each, verify `skills-learned/<skill-name>/SKILL.md` exists. Flag broken wikilinks.
8. **Cross-reference accuracy** — scan for specific event IDs, bug numbers, or timestamps. Verify referenced events actually occurred. Flag stale or fabricated cross-references.

### Phase C — Reconcile and Update
9. **Gather parallel auditor results** — merge findings from all parallel auditors. Deduplicate identical issues.
10. **Generate fix patch** — for each flagged issue, produce a concrete patch (diff or rewrite). Apply patches to SKILL.md.
11. **Update status after fix** — after all fixes applied, set `status: active` in frontmatter. If any issue remains unresolved, set `status: draft` and log to `#⚙️系統`.

## Pitfalls

- **Truncation at save time** — skills generated by skill-reviewer may be silently truncated (e.g., step 7 cut off mid-sentence). Always verify file size ≥ expected and last line is complete. If truncated, patch and re-save.
- **Duplicate completion events** — the runtime may emit two duplicate `completion` events for the same task. Ignore the second; do not double-count or re-process.
- **Stray scaffolding text before `---`** — frontmatter delimiters may have extra text above them. Always verify `---` appears as the first line of the frontmatter block.
