## 反思沉澱系統（核心創新）⭐

```
交易完成 → AI自動複盤 → 提取策略模板 → 評分 → 入規則庫
```

**我哋可以點用：**

| 他們 | 我們可以借鑒 |
|------|-------------|
| 每筆交易後自動分析 | 每個 Session 後自動總結 "學到乜" |
| 成功案例 → 策略模板 | Error 修復後 → 記入 errors.json + 解決方案模板 |

## 評分公式（值得參考）

佢嘅策略評分：
```
策略得分 = (勝率×0.4) + (收益率×0.4) + ((1-風險評分)×0.2)
```

**我哋可以變成：**
```
Agent/Skill 評分 = (成功率×0.4) + (效率×0.3) + (穩定性×0.3)
```

可以用喺：
- 評估邊個 model (MiniMax vs Kimi vs Ollama) 表現更好
- 評估邊個 Skill 最有價值
- 評估 Router 決策準確度

## 🔑 核心問題：自動檢索機制

### 目前問題
- Success/Failure 分類寫咗，但**我唔會主動consult**
- 每次答嘢靠 context + MEMORY.md，**冇自動檢索相關例子**
- 歷史經驗沉澱咗，但**沉嘅地方我唔會主動去撈**

### 理想效果
```
你問問題 → 我自動檢索相關例子
  ├─ 成功例子 → 「可以點做」
  └─ 失敗例子 → 「避免咩坑」
```

### 需要實現
1. **例子分類系統** - 成功/失敗案例明確tagged
2. **快速檢索接口** - `memory_search` 同類例子
3. **答嘢前自動觸發** - 我答之前先問自己：「過去有冇類似情況？」

### 具體做法
- 現有 `errors.json` 已有失敗案例
- 需要新增 `success_patterns.json` 成功案例
- 整合到 `memory_search` 機制
