## Description

根據 AlexZ 🦀 既「Agent 設計的核心約束與原則」文章，將以下架構整合到現有系統：

### 物理約束 (不可違抗的現實)
| 約束 | 說明 |
|------|------|
| C1 | 上下文窗口有限且昂貴 (128K token = 300頁書) |
| C2 | KV Cache 前綴一致性 (匹配 $0.30/M，不匹配 $3.00/M) |
| C3 | 注意力在中間衰減 |
| C4 | LLM 輸出不可靠 (會循環、會幻覺、會忘記) |
| C5 | 工具執行不可預測 |
| C6 | 信息有不同的半衰期 |

### 設計原則 (P1-P10)
| 原則 | 核心內容 | 優先級 |
|------|---------|--------|
| P1 | KV Cache 前綴一致性 - System prompt 構建後永遠唔變 | 最高 |
| P2 | Append-Only 上下文 - 只能追加，唔可以插入/重排 | 2 |
| P3 | 按需加載一切 - 唔預裝任何可以推遲既信息 | 6 |
| P4 | 漸進降級 - 三階段 Pruning | 5 |
| P5 | 末尾復述 - 關鍵狀態必須複述到末尾 | 7 |
| P6 | 系統級不信任 - 獨立於 LLM 既防護機制 | 4 |
| P7 | 可中斷性 - 用戶可以隨時中斷 | 3 |
| P8 | 信息只轉換不消失 - pruning 既內容要可檢索 | 8 |
| P9 | 可觀測性優先 - 每個關鍵路徑都要有指標 | 9 |
| P10 | 文件系統即記憶 - 用文件读写管理工作狀態 | 最低 |

### 優先級順序
```
P1 > P2 > P7 > P6 > P4 > P3 > P5 > P8 > P9 > P10
```
