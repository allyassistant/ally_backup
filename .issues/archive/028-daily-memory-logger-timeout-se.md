## 問題
Daily Memory Logger cron job 每 6 小時 timeout

## 原因
- Session File太大 (9MB)
- 讀取時間 > 60 秒 (timeout limit)
- Cron job 每 6 小時嘗試讀取呢個檔案
- 每次都 timeout
- Tool results 重複包含 setup files (MEMORY.md 被讀取 62 次, AGENTS.md 39 次)

## 受影響既 Session
- 1a65ffc4-5000-487e-aead-246405056f6a.jsonl (9MB, 3月6-7號內容)
- 2c0d854e-17e3-4fa7-9a79-3bd03f779576.jsonl (1.7MB)
- 2fc946cc-bc11-47be-bc21-4dcd19830fed.jsonl (3.4MB)

## 解決方案 (已實施)

### 1. 用 tail 讀取最後 20 行
- **改動**: 用 `tail -n 20` 代替 `fs.readFileSync` 讀取成個 file
- **效果**: 9MB 檔案變成讀取最後 ~50KB，幾秒搞掂

### 2. Content Filters (Skip Patterns)
- **改動**: 加入 skip patterns 過濾重複既 setup files 同 metadata

| 類別 | Filter Patterns |
|------|------------------|
| **Setup Files** | `# SOUL.md`, `# AGENTS.md`, `# MEMORY.md`, `# USER.md`, `# TOOLS.md`, `# HEARTBEAT.md`, `# IDENTITY.md`, `# BOOTSTRAP.md` |
| **Session 啟動** | `Session Startup sequence`, `Execute your Session Startup` |
| **原有 patterns** | `untrusted metadata`, `conversation_info`, `message_id`, `sender_id`, `system:`, `/Users/ally/.openclaw`, `===`, `Process exited`, `HEARTBEAT_OK`, `NO_REPLY` |

### 3. Size Check
- **改動**: Skip >50MB 既 extreme files
- **原因**: tail 可以處理 1-50MB，但超大檔案可能有問題
