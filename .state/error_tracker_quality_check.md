### 🚨 P0 Issue: Async/Sync 混用

**問題函數：** `scan()`, `list()`, `stats()`

```javascript
// error_tracker.js
async function scan() {
  // ...
  unique.forEach(addError);  // addError() 內呼叫 saveErrors() → fs.writeFileSync()
  // ...
}
```

**影響：**
- `async function` 內呼叫 `addError()` → `saveErrors()` → `fs.writeFileSync()`
- 會阻塞 event loop
- 不是災難性問題（blocking write 很快），但違反最佳實踐

**建議：**
將 `saveErrors()` 改為 async，使用 `fs.promises.writeFile()` + `fs.promises.rename()`

**檔案：** `scripts/error_tracker.js`

**修改方向：**
```javascript
// 將 saveErrors 改為 async
async function saveErrors(data) {
  const tmpFile = ERRORS_JSON + '.tmp';
  try {
    // ...
    await fs.promises.writeFile(tmpFile, JSON.stringify(data, null, 2));
    await fs.promises.rename(tmpFile, ERRORS_JSON);
  } catch (err) { ... }
}

// 然後更新所有呼叫者

  // ...
  await saveErrors(data);  // 加 await
```
