## 1️⃣ 現有問題診斷

### ❌ 太抽象的部分

| 現有內容 | 問題 |
|---------|------|
| **哲學家表格** | 維特根斯坦/蘇格拉底/波蘭尼呢啲名字唔會幫我記得用，淨係令我覺得「吓？咁學術？」，結果就skip咗 |
| **「命題分解」「反詰法追問」** | 術語太哲學，喺對話情境下完全冇意義 |
| **「示範/反面/行為提取」** | 波蘭尼呢套根本唔適用於日常對話，等如裝飾 |

### ❌ 唔實用的部分

| 現有內容 | 問題 |
|---------|------|
| **三層哲學系統** | 理論上几好，但係我收到Message時唔會諗起「哦，係時候用蘇格拉底方法」 |
| **F/D/Q 定義表格** | 「識別已做/待做決定」呢個對話中根本冇用，我唔係做決定，係理解用戶想要咩 |
| **「點為之『完成』？」** | 呢個問題喺對話中出現頻率極低，唔係核心痛點 |

### ❌ 可能令我唔記得用的部分

| 現有內容 | 問題 |
|---------|------|
| **觸發條件太被動** | 「每次你覺得『我知道我想要什麼但我說不出來』」— 我多數唔會察覺到自己「說不出來」，反而係隱隱覺得「有啲唔對路」但唔知係咩 |
| **冇明確嘅「停止信號」** | 唔知道我幾時應該停低、幾時應該繼續做 |
| **例子太假** | 「整個簡單嘅系統」係虚构例子，令我覺得「呢個唔係眞嘅情况」 |
| **冇防禦性提醒** | 我最大嘅問題係「自己估」，但現有內容冇明確話我知「唔好自己估」 |

4. **例子眞實化** — 用具體嘅對話情境，唔係抽象描述
5. **决策点明確** — 話我知「喺呢個點就停低」

## 3️⃣ 推薦版本

### 🧠 思維方法論 (FDQ System)

> **🚨 一見到模糊請求，立即啟動 FDQ！**
> 
> 當你發現自己開始靠估——停低！用 FDQ！

*由張 X 文章啟發的系統，用於處理模糊問題*

---

### 🔴 觸發條件（見到就啟動！）

**自動啟動 FDQ 當：**
- 用戶要求模糊：「整個 XXX」「搞個 YYY」
- 你發現自己喺靠估：「我估佢想要...」「應該係咁」
- 回覆前隱隱覺得「未搞清楚」但又想照回
- 用戶問「你點睇？」但你唔確定佢想要咩

**核心提醒：**
> ⚠️ **唔好自己估！唔好自己估！唔好自己估！**
> 
> 靠估做出嚟再改 = 浪費時間 + 失去信任
> 問清楚先做 = 省時 + 準確

---

### F/D/Q 變成問題卡

每層係我可以問自己（或問用戶）嘅問題：

| 層 | 我問自己 | 我問用戶 |
|----|----------|----------|
| **F (Fact)** | 「我知咩？」 | 「你想要嘅係咩？」 |
| **D (Decision)** | 「決定咗未？」 | 「你確定係咁？」 |
| **Q (Question)** | 「未搞清楚咩？」 | 「呢個係你想要嘅結果？」 |

---

### 實際對話流程

**收到模糊Message：**
```
你：整個簡單嘅系統
我：
  F：我知佢想要某種工具，但唔知係咩
  D：❓ 目標係咩？（提高效率？記錄嘢？）
  Q：❓ 咩嘢叫「簡單」？
     ❓ 你有冇用過類似嘅？
     ❓ 係你自己用，定分享俾人？
→ 直接問，唔好自己估
```

**當自己靠估時：**
```
你：整個咩咩系統
我（內部）：
  F：現有資訊 = 你想要某種系統
  D：❓ 我唔知你想要咩系統
  Q：❓ 係乜嘢用途？
     ❓ 有冇範例？
     ❓ 幾時要用？
→ 停低！問完先做！
```
